1. プロジェクト概要

目的

大分の小規模企業（建設・不動産・士業など）向けに、請求書・見積書作成業務を効率化するWebアプリケーションを提供する。

クライアント専用型の優位性
  • 1社ごとに独立したシステムを納品（マルチテナントではない）
  • カスタマイズが自由（大手SaaSとの差別化）
  • データ完全分離でセキュリティが高い
  • 各社の要望に柔軟に対応可能

ターゲット顧客

  • 建設業: 見積書・請求書作成、工事進捗管理
  • 不動産業: 物件情報管理、問い合わせ対応
  • 士業: 顧客管理、書類作成
  • 小売・飲食: 在庫管理、予約管理

価格設定

  • 初期費用: 50,000円（開発・導入・レクチャー）
  • 月額費用: 10,000円（保守・カスタマイズ対応）

3. 技術スタック

項目 技術 理由
バックエンド PHP 8.0+ yojitu.comと同じ、保守性が高い
データベース SQLite（開発・小規模）
MySQL（本番・大規模）		 SQLiteは導入が簡単、スケール時にMySQL移行
PDF生成 TCPDF または mPDF 日本語対応、カスタマイズ性が高い
フロントエンド Vanilla JS（jQuery不使用） 軽量、シンプル
デザイン yojitu.com base.css流用 ナチュラルブラウン系、無印良品風
認証 セッション認証（Phase 2） シンプル、小規模向け

4. データベース設計

4.1 テーブル一覧

customers（顧客テーブル）

カラム名 型 制約 説明
id INTEGER PRIMARY KEY 顧客ID（自動採番）
name TEXT NOT NULL 顧客名
postal_code TEXT 郵便番号（7桁）
address TEXT NOT NULL 住所
phone TEXT 電話番号
email TEXT メールアドレス
notes TEXT 備考
created_at DATETIME DEFAULT CURRENT_TIMESTAMP 登録日時
updated_at DATETIME 更新日時

invoices（請求書テーブル）

カラム名 型 制約 説明
id INTEGER PRIMARY KEY 請求書ID
invoice_number TEXT UNIQUE NOT NULL 請求書番号（INV-20250129-001）
customer_id INTEGER NOT NULL 顧客ID（外部キー）
issue_date DATE NOT NULL 発行日
due_date DATE NOT NULL 支払期限
subject TEXT 件名
subtotal INTEGER NOT NULL 小計（税抜）
tax INTEGER NOT NULL 消費税
total INTEGER NOT NULL 合計（税込）
bank_info TEXT 振込先情報
notes TEXT 備考
status TEXT DEFAULT 'draft' ステータス（draft/sent/paid）
pdf_path TEXT PDF保存パス
is_deleted INTEGER DEFAULT 0 論理削除フラグ
created_at DATETIME DEFAULT CURRENT_TIMESTAMP 作成日時
updated_at DATETIME 更新日時

invoice_items（請求書明細テーブル）

カラム名 型 制約 説明
id INTEGER PRIMARY KEY 明細ID
invoice_id INTEGER NOT NULL 請求書ID（外部キー）
item_name TEXT NOT NULL 品目名
quantity INTEGER NOT NULL 数量
unit_price INTEGER NOT NULL 単価
amount INTEGER NOT NULL 金額（quantity × unit_price）
sort_order INTEGER DEFAULT 0 表示順

quotes（見積書テーブル）

請求書テーブルとほぼ同じ構造。違いは以下：

  • quote_number: 見積書番号（QUO-20250129-001）
  • valid_until: 有効期限（DATE型）
  • delivery_date: 納期（DATE型、任意）
  • status: draft/sent/accepted/rejected

settings（設定テーブル）

カラム名 型 説明
key TEXT 設定キー（例: company_name, tax_rate）
value TEXT 設定値

5. 機能仕様

5.1 ダッシュボード Phase 1

表示内容

  • 今月の統計: 請求書発行件数、請求書合計金額、見積書発行件数
  • 未入金リスト: 支払期限が過ぎた請求書（赤色）、支払期限が近い請求書（黄色）
  • 最近の活動: 直近10件の請求書・見積書

5.2 顧客管理 Phase 1

5.2.1 顧客一覧

  • 表示項目: 顧客ID、顧客名、住所、電話番号、メールアドレス、登録日
  • 検索機能: 顧客名で部分一致検索
  • ソート: 登録日、顧客名
  • ページネーション: 1ページ20件

5.2.2 顧客追加

  • 入力項目: 顧客名（必須）、郵便番号、住所（必須）、電話番号、メールアドレス、備考
  • バリデーション:
    • 顧客名: 100文字以内
    • 郵便番号: 7桁（ハイフンなし）
    • メール: RFC準拠

5.2.3 顧客編集

  • 既存情報の編集
  • 削除機能（請求書紐付けチェック）

5.3 請求書管理 Phase 1

5.3.1 請求書一覧

  • 表示項目: 請求書番号、顧客名、請求日、支払期限、合計金額、ステータス、作成日
  • 検索: 顧客名、請求書番号
  • フィルター: ステータス（未送付/送付済/入金済）、期間
  • ソート: 請求日、金額
  • CSV出力

5.3.2 請求書作成

ステップ1: 基本情報

  • 顧客選択（プルダウン）
  • 請求日（デフォルト: 今日）
  • 支払期限（デフォルト: 請求日+30日）
  • 件名（例: 2025年1月分 Web制作費用）

ステップ2: 明細入力

  • 品目（テキスト、最大50文字）
  • 数量（数値）
  • 単価（数値）
  • 金額（自動計算: 数量 × 単価）
  • 明細行の追加/削除（最大20行）

ステップ3: 計算

  • 小計（自動計算）
  • 消費税（10%、自動計算）
  • 合計（自動計算）

ステップ4: 備考・その他

  • 振込先情報（テンプレート選択可）
  • 備考（自由記入）

5.3.3 請求書プレビュー・保存

  • PDF形式でプレビュー表示
  • 修正ボタンで編集画面に戻る
  • 保存時:
    • データベースに保存
    • PDF自動生成（/pdfs/invoices/INV-YYYYMMDD-001.pdf）
    • ステータス: draft（未送付）

5.4 見積書管理 Phase 1

請求書とほぼ同様の機能。以下の違いあり：

  • 見積書番号: QUO-YYYYMMDD-001
  • 有効期限（デフォルト: 作成日+30日）
  • 納期（任意入力）
  • ステータス: draft/sent/accepted/rejected

5.5 設定 Phase 1

  • 会社情報編集（社名、住所、電話、メール）
  • 振込先情報編集（複数登録可）
  • 消費税率設定（デフォルト: 10%）
  • ロゴ画像アップロード

5.6 メール送信 Phase 2

  • 件名: 【請求書】○月分のご請求（株式会社○○様）
  • 本文テンプレート
  • PDF添付
  • 送付後ステータス: sent（送付済）

5.7 その他機能 Phase 2

  • 見積書→請求書変換
  • ユーザー認証（ログイン機能）
  • データバックアップ（SQLiteファイルダウンロード）
  • CSV一括エクスポート

6. Phase分け

Phase 1（MVP: 最初に作る）優先

目標: 最小限の機能で請求書・見積書作成を実現し、営業資料として使える状態にする
  • ✅ ダッシュボード（基本統計）
  • ✅ 顧客管理（追加・編集・一覧）
  • ✅ 請求書作成・一覧・PDF出力
  • ✅ 見積書作成・一覧・PDF出力
  • ✅ 設定（会社情報・振込先）

工数見積もり: 10〜15時間

Phase 2（追加機能）次フェーズ

目標: 実際の運用で必要になる機能を追加
  • ⏳ メール送信機能
  • ⏳ 見積書→請求書変換
  • ⏳ ユーザー認証（ログイン機能）
  • ⏳ データバックアップ
  • ⏳ CSV一括インポート・エクスポート

工数見積もり: 5〜10時間

運用方針
  • Phase 1を完成させて、営業資料・デモとして使用
  • 1社目の顧客獲得後、実際の運用で必要な機能をPhase 2で追加
  • 各社の要望に応じて個別カスタマイズ
