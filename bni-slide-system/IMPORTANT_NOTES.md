# 重要な注意事項

## ベーシック認証について

### 現在の仕様
**このシステムにはベーシック認証を設定していません。**

- ユーザー登録ページ（register.php）は誰でもアクセス可能
- その他のページも認証不要でアクセス可能

### 過去の問題
以前、.htaccessにベーシック認証を追加したところ、以下の問題が発生しました：

1. register.phpにもベーシック認証がかかってしまった
2. 新規ユーザーが登録できなくなった
3. 特定ファイルのみ認証から除外する設定が複雑だった

### 対応履歴
- コミット `604b58e` でユーザー登録機能を実装
- その後、誤ってベーシック認証を.htaccessに追加
- コミット `aa5246b` で.htaccessを `604b58e` 時点に戻して解決

### 今後の方針
**ベーシック認証は設定しない**

もし将来的にセキュリティが必要な場合は：
- 管理画面（admin/）のみに認証を設定
- ユーザー登録ページ（register.php）は常に認証不要にする
- PHPセッションベースの認証を検討

### .htaccessの変更時の注意
.htaccessを変更する際は、以下を確認してください：

1. register.phpにアクセスできるか
2. api_register.phpにアクセスできるか
3. 新規ユーザー登録が正常に動作するか

### 参考コマンド
```bash
# 現在の.htaccessの状態を確認
git show HEAD:.htaccess

# 604b58e時点の.htaccessに戻す（必要な場合）
git show 604b58e:.htaccess > .htaccess
git add .htaccess
git commit -m "Revert: .htaccessを604b58e時点に戻す"
git push origin main
```

## ユーザー登録フロー

### 登録ページ
- URL: https://yojitu.com/bni-slide-system/register.php
- 認証: 不要（誰でもアクセス可能）

### 入力項目
1. 名前（必須）
2. メールアドレス（必須・ログインIDとして使用）
3. 電話番号（任意）

### 自動生成される情報
- パスワード: 10文字のランダム英数字
- ユーザー名: メールアドレスをそのまま使用

### 登録後の流れ
1. ウェルカムメールが送信される
2. メールにログイン情報（メールアドレス・パスワード）が記載
3. ユーザーはログイン後、プロフィール画面でパスワード変更可能

## 管理者向け

### ユーザー管理画面
- URL: https://yojitu.com/bni-slide-system/admin/users.php
- 機能: ユーザー一覧表示、登録日・最終更新日確認

### データファイル
- ユーザー情報: `data/members.json`
- 認証情報: `.htpasswd`（現在は使用していない）

### テストユーザー
存在する場合のログイン情報：
- ログインID: yamada@example.com
- パスワード: test123
