実装方法

1. RSSフィードからニュースを取得

function fetchNewsFromRSS() { // 官公庁・業界団体のRSSフィード const rssUrls = [ 'https://www.meti.go.jp/rss/index.rdf', // 経済産業省 'https://www.nta.go.jp/rss/index.rdf' // 国税庁 ]; const allNews = []; rssUrls.forEach(function(rssUrl) { const xml = UrlFetchApp.fetch(rssUrl).getContentText(); const document = XmlService.parse(xml); const root = document.getRootElement(); // RSS 2.0形式の場合 const items = root.getChild('channel').getChildren('item'); items.forEach(function(item) { const title = item.getChild('title').getText(); const link = item.getChild('link').getText(); const pubDate = item.getChild('pubDate').getText(); allNews.push({ title, link, pubDate }); }); }); return allNews; }

スクレイピング対象例

  • 官公庁サイト（経済産業省、厚生労働省、国税庁など）
  • 業界団体のニュースページ（商工会議所、業界団体など）
  • 補助金・助成金情報サイト（ミラサポplus、J-Net21など）
  • 法改正情報サイト（官報、e-Gov法令検索など）

スクレイピングの法的注意点

必ず守るべきルール
  • 利用規約を確認：スクレイピング禁止のサイトは絶対に対象外にする
  • robots.txtを遵守：クローリング禁止の指定に従う
  • アクセス頻度を制限：最低1秒間隔、推奨は3〜5秒間隔
  • User-Agentを設定：正しいUser-Agent文字列を設定する
  • 営業時間外に実行：サーバー負荷を考慮して深夜・早朝に実行
  • 公式APIを優先：APIが提供されている場合は必ずAPIを使用
安全にスクレイピングするために
  • スクレイピング実行前に必ず利用規約を確認
  • アクセス間隔をUtilities.sleep(3000)で3秒空ける
  • 必要最小限のデータのみ収集
  • データの二次利用は慎重に（著作権に配慮）
  • 過度なアクセスは偽計業務妨害罪に該当する可能性あり
