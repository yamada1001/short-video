# Claude Code 作業ルール

**作成日**: 2025-12-11
**最終更新**: 2025-12-11

## 📋 重要なルール

### 1. AskUserQuestion の使用義務
- **必ず毎回**ユーザーに選択肢を提示する際は `AskUserQuestion` ツールを使用すること
- ユーザーが判断すべき事項について、勝手に決めないこと
- 実装方針、確認方法、次のタスクなど、全てAskUserQuestionで確認すること

### 2. このルールの徹底
- このルールは**絶対**に守ること
- セッション再開時には必ずこのファイルを読むこと
- ルール違反があった場合は即座に修正すること

---

## ✅ Good Examples（良い例）

### 例1: 次のタスクを確認する場合
```
❌ 悪い例: 「次はビデオ対応を実装しますね」
✅ 良い例: AskUserQuestionで「次にどのタスクを進めますか？」と選択肢を提示
```

### 例2: 実装方針を決める場合
```
❌ 悪い例: 「YouTube URLのみ対応します」
✅ 良い例: AskUserQuestionで「ビデオのアップロード方法は？」と選択肢を提示
```

### 例3: 確認方法を決める場合
```
❌ 悪い例: 「本番環境で確認しましょう」
✅ 良い例: AskUserQuestionで「どの環境で確認しますか？」と選択肢を提示
```

---

## 📝 ドキュメント化のルール

### 重要な決定事項は必ずドキュメント化
- ユーザーの回答内容
- 実装方針
- 作業進捗

### ドキュメント化する場所
- `WORK_LOG_YYYY-MM-DD.md` - 日々の作業ログ
- `PROGRESS_TRACKING.md` - 進捗トラッキング
- `SESSION_RESUME.md` - セッション再開用ガイド

---

## 🔄 セッション再開時のチェックリスト

1. ✅ `CLAUDE_CODE_RULES.md` を読む（このファイル）
2. ✅ `PROGRESS_TRACKING.md` で現在の進捗を確認
3. ✅ `WORK_LOG_YYYY-MM-DD.md` で最新の作業内容を確認
4. ✅ `git status` でブランチと変更状況を確認
5. ✅ AskUserQuestionで次のタスクを確認

---

## 💡 その他の重要な注意事項

### Gitコミットについて
- コミットメッセージは日本語でOK
- 必ずBNI公式フォーマットを使用（末尾に「🤖 Generated with Claude Code」を追加）

### ファイル編集について
- 既存ファイルはできるだけ Edit ツールを使用
- 新規ファイルのみ Write ツールを使用

### エラーハンドリング
- エラーが発生した場合は必ずユーザーに報告
- 自己判断で修正せず、AskUserQuestionで確認
